   (`typeshed <https://github.com/python/typeshed/blob/5632132/stdlib/builtins.pyi#L2205>`__)

.. exception:: CancelledError
               CancelledError(cls: type[asyncio.exceptions.CancelledError], *args: typing.Any, **kwds: typing.Any) -> asyncio.exceptions.CancelledError

   (`typeshed <https://github.com/python/typeshed/blob/5632132/stdlib/asyncio/exceptions.pyi#L24>`__)

   The operation has been cancelled.

   This exception can be caught to perform custom operations
   when asyncio Tasks are cancelled.  In almost all situations the
   exception must be re-raised.

   .. versionchanged:: 3.8

      :exc:`CancelledError` is now a subclass of :class:`BaseException` rather than :class:`Exception`.


.. exception:: InvalidStateError
               InvalidStateError(cls: type[asyncio.exceptions.InvalidStateError], *args: typing.Any, **kwds: typing.Any) -> asyncio.exceptions.InvalidStateError

   (`typeshed <https://github.com/python/typeshed/blob/5632132/stdlib/asyncio/exceptions.pyi#L31>`__)

   Invalid internal state of :class:`Task` or :class:`Future`.

   Can be raised in situations like setting a result value for a
   *Future* object that already has a result value set.


.. exception:: SendfileNotAvailableError

   (`typeshed <https://github.com/python/typeshed/blob/5632132/stdlib/asyncio/exceptions.pyi#L32>`__)

   The "sendfile" syscall is not available for the given
   socket or file type.

   A subclass of :exc:`RuntimeError`.


.. exception:: IncompleteReadError

   (`typeshed <https://github.com/python/typeshed/blob/5632132/stdlib/asyncio/exceptions.pyi#L34>`__)

   The requested read operation did not complete fully.

   Raised by the :ref:`asyncio stream APIs<asyncio-streams>`.

   This exception is a subclass of :exc:`EOFError`.
